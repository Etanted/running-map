# PRD (Product Requirements Document)
# 러닝맵 (Running Map) — 마라톤 대회 정보 플랫폼

**버전**: 1.0  
**작성일**: 2026년 3월 17일  
**상태**: MVP 완성, v2 기획 중  

---

## 1. 제품 개요

### 1.1 서비스 소개

**러닝맵**은 국내 마라톤·트레일런 대회 정보를 지도 기반으로 한눈에 탐색할 수 있는 웹 플랫폼이다. 대회 위치·일정·코스 정보를 시각화하여 러너들이 참가 대회를 쉽게 찾고 준비할 수 있도록 돕는다.

### 1.2 핵심 가치

| 가치 | 설명 |
|------|------|
| **탐색 편의성** | 지도·리스트·캘린더 세 가지 뷰로 원하는 방식으로 대회 탐색 |
| **코스 정보 시각화** | GPX 기반 코스 경로·고도 프로파일 제공으로 사전 준비 지원 |
| **페이스 계획** | 자체 페이스 계산기로 레이스 전략 수립 지원 |
| **모바일 최적화** | 훈련 중·이동 중 폰으로도 정보 접근 가능 |

### 1.3 목표 사용자

- **주 타깃**: 마라톤·트레일런 대회 참가를 계획 중인 러너 (20~50대)
- **부 타깃**: 대회를 처음 접하려는 입문 러너, 특정 지역/코스 유형을 선호하는 베테랑 러너

### 1.4 프로덕션 URL

- **서비스**: https://running-map-mu.vercel.app
- **GitHub**: https://github.com/Etanted/running-map

## 3. 핵심 데이터 모델

### 3.1 대회 (Event)

```
id, name, date, startTime, venue, address
lat, lng                       위치 좌표
region                         17개 시·도
organizer, hostOrganization    주최·주관
officialWebsite                공식 홈페이지
registrationStart/End          접수 기간
registrationStatus             open | upcoming | closed | unknown
courses[]                      코스 목록
maxParticipants, description, tags
```

### 3.2 코스 (Course)

```
id, eventId, name              "풀코스" | "하프" | "10km" 등
distanceKm, isLongest
startTime, timeLimitMinutes
entryFee                       참가비 (원)
type                           road | trail | crosscountry | track | mixed
gpxData                        GPX XML 문자열
courseImageUrl, routeCoordinates
elevationData, elevationStats  고도 데이터 (상승·하강·최저·최고)
```

### 3.3 필터 상태 (FilterState)

```
region      'all' | 17개 시·도
courseType  'all' | road | trail | crosscountry | track | mixed
status      'all' | open | upcoming | closed | unknown
```

### 4.4 대회 상세 모달

#### 4.4.1 개요
마커·리스트 카드·캘린더 등 어디서든 대회를 선택하면 표시되는 상세 정보 패널.

#### 4.4.2 기능 요구사항

**3탭 구조**

| 탭 | 내용 | 우선순위 |
|----|------|---------|
| **코스 정보** | 최장 코스: 코스 지도(GPX 미니맵 or 이미지) + 고도 프로파일 + 출발·제한·참가비 + 공식 홈페이지 링크 | P0 |
| **코스 목록** | 전체 코스 카드 목록 (거리, 출발, 제한, 참가비). 클릭 시 개별 코스 상세 표시 | P0 |
| **기본 정보** | 대회명, 날짜(요일), 시간, 장소, 주소, 주최·주관, 최대 인원, 접수 기간, 설명 | P0 |

**UI 동작**

| 환경 | 동작 |
|------|------|
| 데스크톱 | 화면 중앙 팝업, `max-w-2xl`, `max-h-[90vh]` |
| 모바일 | 하단에서 슬라이드업 (Bottom Sheet), `h-[92svh]`, 상단 드래그 핸들 표시 |

#### 4.4.3 고도 프로파일 차트

- GPX 데이터 존재 시 Recharts `AreaChart` 렌더링
- 최대 120개 포인트 다운샘플링
- X축: 누적 거리(km), Y축: 고도(m)
- 호버 툴팁: 해당 지점 거리·고도 표시
- 통계 표시: 총 상승(m), 총 하강(m), 최저/최고 고도

#### 4.4.4 GPX 미니맵

- Leaflet + OpenStreetMap 기반
- 빨강 Polyline으로 코스 경로
- 출발점 S(초록) / 도착점 F(파랑) 마커
- 자동 경계 조정 (fitBounds)

### 4.5 페이스 계산기 (`/pace`)

#### 4.5.1 개요
목표 기록 또는 기준 기록을 입력해 km별 스플릿 타임과 레이스 전략을 계산하는 도구.

#### 4.5.2 기능 요구사항

**목표 기록 입력**

| ID | 기능 | 설명 | 우선순위 |
|----|------|------|---------|
| P-01 | 거리 프리셋 | 5km / 10km / 하프(21.1km) / 풀(42.195km) 버튼 | P0 |
| P-02 | 거리 직접 입력 | 임의 거리(km) 숫자 입력 | P0 |
| P-03 | 목표 시간 입력 | HH:MM:SS 형식 | P0 |
| P-04 | 페이스 전략 | 균등 / 초반 가속(Negative Split) / 후반 가속(Positive Split) | P0 |
| P-05 | 스플릿 테이블 | km별 구간 페이스·시간·누적 시간 | P0 |
| P-06 | 전반/후반 비교 | 전반부 vs 후반부 소요 시간 + 비율 바 | P0 |
| P-07 | 대시보드 카드 | 평균 페이스·총 시간·목표 거리·전략명 요약 | P0 |

**Riegel 예측 공식**

| ID | 기능 | 설명 | 우선순위 |
|----|------|------|---------|
| P-08 | 기준 기록 입력 | 5km/10km/하프/풀 중 선택 또는 직접 입력 | P0 |
| P-09 | 예측 거리 선택 | 목표 거리 4가지 프리셋 | P0 |
| P-10 | 예측 결과 표시 | 예상 완주 기록·평균 페이스 출력 | P0 |

**Riegel 공식**: $ T_2 = T_1 \times \left(\frac{D_2}{D_1}\right)^{1.06} $

## 10. 용어 정의

| 용어 | 정의 |
|------|------|
| **GPX** | GPS Exchange Format. GPS 경로 데이터 XML 파일 |
| **스플릿 타임** | 구간별(km별) 통과 예상 시간 |
| **Negative Split** | 후반부를 전반부보다 빠르게 달리는 전략 |
| **Positive Split** | 전반부를 후반부보다 빠르게 달리는 전략 |
| **Riegel 공식** | 기준 기록으로 다른 거리의 예상 기록을 계산하는 공식 |
| **고도 프로파일** | 코스의 거리에 따른 고도 변화를 나타낸 그래프 |
| **접수 상태** | open(접수중) / upcoming(예정) / closed(마감) / unknown |
| **최장 코스** | 한 대회의 여러 코스 중 가장 거리가 긴 코스 |
